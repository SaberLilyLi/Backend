## 智能个人知识库助手 - 后端需求与实现说明

### 一、项目概述

- **项目名称**: 智能个人知识库助手（后端服务）
- **技术栈**: Node.js (Express) + MongoDB + JWT 认证
- **定位**: 作为前端 Vue3+TS 项目的后端支持，为个人知识库提供用户、文档、笔记、标签、分类、语义搜索、知识图谱和统计分析等能力。

### 二、业务目标与当前实现范围

- **已覆盖目标**:
  - **基础知识库管理**: 文档、笔记的 CRUD，标签与分类管理。
  - **用户体系**: 注册、登录、登出，基于 JWT 的私有知识库隔离。
  - **AI 增强基础能力**: 简化版 RAG 语义搜索接口（占位实现，支持后续接入 OpenAI / 本地模型）。
  - **数据可视化支撑**: 概览统计、时间趋势统计接口，供前端做图表展示。
  - **知识图谱支撑**: 基于“文档-标签-分类”的知识图谱数据接口，供前端图谱可视化。

- **待后续扩展（未在当前后端中实现）**:
  - 团队协作与复杂权限体系（目前仅支持个人级隔离）。
  - 语音输入、图片 OCR、多模态内容处理。
  - 多模型管理、插件化 AI 能力扩展。
  - 更精细的版本控制、数据导入导出（当前仅在设计层面）。

### 三、核心数据模型设计（MongoDB）

> 以下仅列出与本项目相关的主要集合，字段略去部分内部实现。

- **User**
  - `username`: 用户名（唯一，校验格式）。
  - `email`: 邮箱（唯一）。
  - `password`: 哈希后的密码。
  - `createdAt`: 创建时间。

- **Document**
  - `title`: 文档标题。
  - `content`: 文档内容（Markdown / 纯文本等）。
  - `content_type`: 内容类型（默认 `markdown`）。
  - `category`: 分类名称（与 Category 逻辑关联）。
  - `tags`: 标签字符串数组（与 Tag 逻辑关联）。
  - `author_id`: 关联用户 ID。
  - `created_at` / `updated_at`: 创建与更新时间。

- **Note**
  - 与 Document 类似，增加 `priority` 字段（优先级）。

- **Tag**
  - `name`: 标签名称。
  - `color`: 颜色值。
  - `owner_id`: 拥有者用户 ID。

- **Category**
  - `name`: 分类名称。
  - `slug`: 唯一标识。
  - `description`: 描述。
  - `icon` / `color`: 展示用字段。
  - `owner_id`: 拥有者用户 ID。

- **Index**（向量索引，占位实现）
  - `documentId`: 关联的文档 ID。
  - `vector`: 文档内容的向量表示（当前为伪向量）。
  - `metadata`: 包含 `title`、`tags`、`category`、`userId` 等。

### 四、功能需求与后端接口映射

#### 1. 用户管理

- **注册 / 登录 / 登出**
  - 接口: 见 `API_DOCUMENTATION.md` 中 **用户认证接口**。
  - 实现: `authController` + `User` 模型 + `JWT`，通过 `middleware/auth` 做权限保护。
  - 需求对齐:
    - 支持邮箱注册登录 ✅
    - 权限管理（个人知识库私有化）✅
    - 更复杂的团队权限 ❌（预留后续扩展）。

#### 2. 知识库管理（文档 / 笔记 / 标签 / 分类）

- **文档管理**
  - 接口: `/api/documents` 系列。
  - 能力: 文档创建、列表查询、单条查询、更新、删除，支持分类和标签过滤、分页。

- **笔记管理**
  - 接口: `/api/notes` 系列。
  - 能力: 与文档类似，增加优先级字段用于个人任务或待办类知识记录。

- **标签管理**
  - 接口: `/api/tags` 系列。
  - 能力: 用户级标签体系，支持颜色、重名校验、增删改查。

- **分类管理**
  - 接口: `/api/categories` 系列。
  - 能力: 用户级分类体系，支持 slug 唯一标识，增删改查。

#### 3. AI 增强功能

- **语义搜索 / 智能问答**
  - 接口: `POST /api/search`
  - 能力:
    - 接收自然语言 `query`。
    - 通过 `aiService` 中的 `semanticSearch`，在 `Index` 集合中基于“伪向量 + 点积”进行相似度检索。
    - 返回相关文档元数据与一个简单的“答案片段”（从最高分文档抽取）。
  - 说明:
    - 当前实现为 **占位实现**，方便前端联调与 UI 展示。
    - 后续可无缝替换为:
      - OpenAI Embeddings / ChatCompletions。
      - 本地向量数据库 + LlamaIndex / LangChain。

- **向量索引维护**
  - 目前实现了 `Index` 模型和 `upsertDocumentIndex` 服务方法，可在后续:
    - 文档创建 / 更新时调用，自动刷新向量索引。
    - 扩展批量重建索引的管理接口。

- **知识图谱**
  - 接口: `GET /api/graph`
  - 能力:
    - 基于当前用户的 Document：
      - 文档节点（`type=document`）。
      - 标签节点（`type=tag`）。
      - 分类节点（`type=category`）。
    - 自动生成 `belongs_to`（文档 -> 分类）和 `has_tag`（文档 -> 标签）边。
  - 用途:
    - 前端可直接基于该接口数据，用 ECharts / AntV G6 渲染知识图谱。
  - 后续扩展点:
    - 支持从内容中抽取实体关系（例如通过 NLP / LLM 自动识别概念、人物、技术等）。

#### 4. 数据可视化与统计

- **概览统计**
  - 接口: `GET /api/analytics/overview`
  - 能力: 返回当前用户的文档总数、笔记总数。

- **使用趋势**
  - 接口: `GET /api/analytics/trends`
  - 能力:
    - 按日期聚合用户创建的文档和笔记数量。
    - 输出可被前端直接用于折线图 / 柱状图的数据结构。

### 五、非功能需求与目前支持情况

- **性能**
  - 按用户隔离，数据量初期有限，MongoDB 查询 + 简单聚合足以满足 DEMO / 简历项目场景。
  - 分页查询已在文档与笔记接口中实现。

- **安全与隐私**
  - JWT 认证中间件保证接口访问需要有效 Token。
  - 不同用户之间，通过 `author_id` / `owner_id` 字段隔离数据访问。
  - 建议在部署时：
    - 使用 HTTPS。
    - 将 `JWT_SECRET`、`MONGODB_URI` 放在环境变量中（`.env`）。

- **可维护性与扩展性**
  - 路由 / 控制器 / 模型 / 中间件 / 服务 分层清晰。
  - 新增 AI 模型时只需扩展 `services/aiService.js` 和少量配置。
